※1 Claude Code / Anthropic API の利用規約に準拠。入力データの取り扱いは Anthropic のプライバシーポリシーに従う。

※2 MAILTO_EMAIL は API レート制限緩和のための polite pool 識別子として使用される。認証・登録は不要。

2. 外部 API への送信内容の詳細

2.1 OpenAlex API

送信データ: 検索クエリ（テーマ・手法キーワード）、MAILTO_EMAIL、DOI（メタデータ補完時）
  • OpenAlex は完全無料・認証不要のオープン学術データベース
  • 送信されるのは検索クエリのみ。研究内容の全文は送信しない
  • MAILTO_EMAIL はレート制限緩和のためのメタデータとして使用（応答メールは来ない）

2.2 CrossRef API

送信データ: DOI、書誌情報クエリ（著者名・年・タイトルヒント）、MAILTO_EMAIL
  • DOI を渡して BibTeX メタデータを取得するのみ
  • 研究内容は送信しない

2.3 Exa MCP（補完検索時のみ）

送信データ: 検索クエリ、Exa API キー
  • OpenAlex でカバレッジ不足の場合のみ起動する（通常は使用しない）
  • API キーは .claude/mcp.json に記載。リモートリポジトリに push しない（.gitignore 推奨）

2.4 Claude Code / Anthropic API

送信データ: 研究内容（input.md の全文）、論文ドラフト（レビュー・執筆時）
  • Claude Code のすべての処理（執筆・レビュー・統合）で研究内容が Anthropic のサーバーに送信される
  • Anthropic のプライバシーポリシー・利用規約に従い処理される
  • 未公開研究・機密情報を含む場合は、Anthropic の商用プランのデータ保護ポリシーを事前に確認すること

2.5 ChatGPT・Gemini（レビュー時・ユーザー操作）

送信データ: 論文ドラフト全文（ユーザーが手動で貼り付ける）
  • CP4 レビュー時にユーザーが手動でドラフトを貼り付ける
  • 各 AI の利用規約（OpenAI / Google の商用利用条件・入力データの扱い）を事前に確認すること
  • 未公開研究データを投稿する場合はとくに注意が必要

2.6 NotebookLM（Factcheck 時・ユーザー操作）

送信データ: 引用論文 PDF（ユーザーがアップロードする）
  • 著作権のある論文 PDF をアップロードする場合は、当該論文の利用規約と所属機関のライセンス条件を確認すること
  • NotebookLM Free 版のソース上限（50件/ノートブック）に注意

3. Git 管理ポリシー（.gitignore）

3.1 リモートに上がらないもの（.gitignore で除外）

対象除外パターン理由
研究プロジェクト全体projects/*（template/ を除く）未公開研究データの保護
個人設定ファイルscripts/config.pyメールアドレスの漏洩防止
SJR CSV データdata/sjr_*.csvファイルサイズ（11MB）・ライセンス
PDF ファイル*.pdf著作権保護
学会テンプレートtemplates/著作権保護
Python キャッシュ__pycache__/、*.pyc不要ファイル

3.2 リモートに上がるもの

フレームワーク本体のみ: .claude/、scripts/（config.example.py を含む）、manuals/、docs/、data/journal_abbr.json、CLAUDE.md、README.md、requirements.txt

公開前に git grep -n -E "email|api_key|password" でスキャンし、機密情報が含まれていないことを確認すること。

4. API キー管理

4.1 Exa API キー

絶対にコミットしないこと。.claude/mcp.json は .gitignore に追加するか、キーは環境変数経由で渡すことを推奨する。
.claude/mcp.json に設定する場合:
{
  "mcpServers": {
    "exa": {
      "env": { "EXA_API_KEY": "<ここに実際のキーを記載>" }
    }
  }
}

※ このファイルを git add する前に必ず確認すること。
推奨: .gitignore に .claude/mcp.json を追記するか、
      EXA_API_KEY を OS の環境変数に設定して参照する。

4.2 MAILTO_EMAIL（メールアドレス）

  • scripts/config.py に記載。このファイルは .gitignore で除外済み
  • 新規セットアップ時は config.example.py をコピーして config.py を作成すること
  • メールアドレスは API のポライトプール識別子として使用されるだけで、漏洩しても直接的なリスクは低い。しかし公開リポジトリへの commit は避けること

5. 著作権・ライセンスの取り扱い

5.1 引用論文 PDF

  • 自動取得する PDF は OpenAlex の OA リンク（オープンアクセス版）のみ使用する
  • 所属機関のライセンスで取得した PDF は pdfs/ に配置するが、.gitignore により自動的にリモートへは上がらない
  • PDF を第三者と共有する場合は、各論文の利用規約を確認すること

5.2 SJR データ

  • Scimago Journal Rank のデータは scimagojr.com から取得
  • 商用利用には別途ライセンス確認が必要な場合がある
  • CSV は data/sjr_*.csv として .gitignore で除外されるため、リポジトリには含まれない

6. 研究データのプライバシー

6.1 未公開研究の取り扱い

注意: /draft・/review-packet 実行時に、研究内容が Claude Code（Anthropic）・ChatGPT（OpenAI）・Gemini（Google）のサーバーへ送信される。査読前・特許出願前の未公開研究を取り扱う場合は、各サービスの商用プランの「データ非学習オプション」や「エンタープライズ契約」の利用を検討すること。

6.2 progress.yaml・checkpoints のバックアップ

  • projects/ は .gitignore で除外されるため、GitHub によるバックアップは行われない
  • 重要な研究プロジェクトは、OneDrive / Google Drive 等のクラウドストレージと併用してバックアップすること

7. セキュリティチェックリスト

#確認項目対応
